# 同步导出公司格式报销单：技术设计

## 1. 目标与边界

在不改变 OCR、分类、重复票和行程校对流程的前提下，为统一导出入口增加公司正式报销单 Excel 与可打印 PDF。现有 `00_报销清单.xlsx` 保持兼容。

现有 `invoice_agent.excel.write_workbook()` 继续只负责清单。新增：

- `invoice_agent/company_reimbursement.py`：数据转换、中文金额大写、模板填充、插行、分页和校验。
- `invoice_agent/pdf_export.py`：Windows Excel COM 能力检测及 PDF 导出。
- `invoice_agent/templates/company_reimbursement_template.xlsx`：清理后的三 Sheet 模板。

## 2. 数据流

1. 公共导出入口先生成现有 JSON 和 `00_报销清单.xlsx`。
2. 过滤 `include_in_amount=True` 的有效票据，行程单不计金额和张数。
3. 独立计算餐补展示记录，并完成差旅、日常和报销明细分类。
4. 复制内置模板，填充并动态分页，生成 `01_公司报销单.xlsx`。
5. 校验总金额、工作表、打印区域和页数。
6. Windows 且安装 Microsoft Excel 时导出合并的 `01_公司报销单.pdf`。
7. 每个产物独立记录成功、跳过或失败。

## 3. 输出契约

建议新增：

```python
@dataclass
class ExportArtifact:
    kind: str
    path: Path
    status: str  # success | skipped | failed
    message: str = ""

@dataclass
class ExportResult:
    status: str  # success | partial_success | failed
    artifacts: list[ExportArtifact]
    warnings: list[str]
```

Web 保留既有 `excel_path`，同时增加公司 Excel、PDF、导出状态、提醒和产物列表。批量 package 使用相同结构。

## 4. 模板处理

内置模板只保留：

- `报销明细表`
- `差旅费报销单`
- `日常费用报销单`

清除示例数据、签名和打印区域外残留，保留正式样式、合并单元格、打印设置和审批栏。

实现优先直接编辑 OOXML，避免 openpyxl 往返保存破坏模板版式。openpyxl 仅用于只读验证。

### 报销明细表

- A4 纵向。
- 按分类定位标题、明细和小计。
- 空间不足时在小计前插行并复制样式。
- 同步更新小计、总计、后续区域、签字区和打印区域。
- 存在无法归类项目时动态增加“其他费用”区域。

### 差旅费报销单

- A5 横向，每页最多 12 条交通明细。
- 超量时复制整张差旅单分页。
- 每页重复基础信息。
- 住宿、餐补、油费、过路费、退改费等汇总项只填第一页。
- 第一页总额包含全部交通页与汇总项，后续页不重复计总额。

### 日常费用报销单

- A5 横向，每页最多 8 条。
- 无数据时删除模板页。
- 按实际数据复制分页，每页只计算本页合计。

## 5. 映射规则

- 城际交通进入“行程交通费”。
- 网约车、出租车进入“市区交通费”。
- 高速通行票进入“过路费”。
- 住宿、油费、退改费和餐补进入差旅单对应项目。
- 招待费、材料费、办公费、礼品礼卡及其他非差旅费用进入日常费用单。
- 日常费用内容依次取人工备注、说明、销方名称、凭证类别，并清理换行。
- 烟酒明确用于赠送或客户维护时归礼品礼卡；招待现场消费归招待费；无法判断时归其他费用并提醒。
- 餐补日期显示出差日期范围，张数为 0。
- 住宿在明细表逐张列出，在差旅单汇总为一个金额；日期优先使用出差起止日期。

## 6. 金额与日期

- 所有金额使用 `Decimal`。
- 公司表总额必须等于类别汇总总额。
- 每张有效发票计一张，行程单与重复票不增加张数。
- 中文金额大写由程序生成固定文本。
- 填报日期使用实际导出日期固定值。
- 项目名、报销人、部门缺失或疑似占位值不阻断导出，只产生提醒。

## 7. PDF

- 仅 Windows 且检测到 Microsoft Excel 时启用。
- 后台打开公司 Excel，通过固定格式导出单个合并 PDF。
- PDF 只包含实际存在的工作表，并保持纸张、方向与分页。
- `finally` 中关闭工作簿、退出 Excel 并释放 COM 对象。
- 不支持时标记跳过；转换失败时标记失败，但不删除 Excel。

## 8. 错误处理

- 清单失败：整体失败，不继续生成公司文件。
- 公司 Excel 失败：保留清单，状态为部分成功。
- PDF 跳过：保留两份 Excel并提示。
- PDF 失败：保留两份 Excel，状态为部分成功。
- 模板缺失、损坏或关键锚点缺失时返回明确错误。

## 9. 验证与回滚

- 校验 Sheet 名称、打印区域、纸张、方向、页数、关键单元格和总金额。
- 使用真实参考数据进行 Web 灰度，并通过 Microsoft Excel 生成 PDF。
- 人工抽查分页、签字栏、金额大写和 A4/A5 打印效果。
- 公司文件通过公共导出入口中的独立调用接入，必要时可关闭该调用，不影响现有清单导出。
